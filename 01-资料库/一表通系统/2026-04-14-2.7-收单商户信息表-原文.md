# 2.7 收单商户信息表

来源：用户提供原文  
性质：监管一表通系统相关原文摘录  
整理日期：2026-04-14

## 采集业务范围

报送本行因线下线上收单业务而拓展的线下 POS 终端、线上终端以及二维码商户的信息。同一商户存在多个 POS 终端的，按多条报送。报数期开始前商户状态为“失效”或“注销”等状态的，不填。

## 数据项清单

| 数据项编码 | 数据项名称 | 数据类别 | 数据格式 | 主键 | 数据项解释 | 监管修改记录 |
| --- | --- | --- | --- | --- | --- | --- |
| B070001 | 商户ID | 编码类 | anc..24 |  | 唯一识别商户的编码。指持卡人实际支付的商户编号，不仅包括银行自有商户编号，第三方支付机构收单对应的交易也需填报商户编号。 |  |
| B070002 | 客户ID | 编码类 | anc..60 |  | 按照既定规则生成用于区分客户的唯一编码。银行机构需保证机构内部针对同一个户有且仅有一个客户ID。其中开户行为他行的商户，其客户ID允许为空。 |  |
| B070003 | 机构ID | 编码类 | anc..24 |  | 按照既定规则生成分配给实际业务经办最小单位机构的唯一编码。 |  |
| B070004 | 商户名称 | 文本类 | anc..450 |  | 商户注册登记的名称。指持卡人实际支付的商户名称。对公客户填报商户注册登记证件上记载的法定名称全称。个人客户若未收集商户注册登记证件信息的，允许填报客户名称。 |  |
| B070005 | 是否为POS机特约商户 | 指示器类 | 1!n |  | 0-否；1-是。若为 POS 商户填报为是，其他情况无 pos 终端的填报为否。 |  |
| B070006 | 终端号 | 编码类 | anc |  | 银行自定义唯一标识的可供交易或查询的机具设备编码。 |  |
| B070007 | 商户类别码 | 编码类 | 4!n |  | 由四位数字代码组成的银联 MCC 码，用于标明银联卡交易环境、所在商户的主营业务范围和行业归属。若无 4 位银联 MCC，可以为空。 |  |
| B070008 | 商户类别码名称 | 文本类 | anc |  | 商户类别码对应的名称。 |  |
| B070009 | 清算卡号或账号 | 编码类 | an |  | 收单商户绑定的收款卡号或账号。 |  |
| B070010 | 清算账号类型 | 代码类 | 2!n |  | 01-本行卡；02-本行对公结算账户；03-他行卡；04-他行对公结算账户；00-其他。 |  |
| B070011 | 清算账户名称 | 文本类 | anc |  | 收单商户绑定的商户收款账户名称。 |  |
| B070012 | 清算账号开户行名称 | 文本类 | anc |  | 填报清算账号所属行名，以金融机构许可证登记名称为准。若为第三方支付平台，填写第三方支付平台名称。 |  |
| B070013 | 商户起效日期 | 日期类 | YYYY-MM-DD |  | POS 特约商户签约的日期，默认值 9999-12-31。若不涉及可置空。 |  |
| B070014 | 商户失效日期 | 日期类 | YYYY-MM-DD |  | POS 特约商户签约的失效或不可用的日期，默认值 9999-12-31。若不涉及可置空。 |  |
| B070015 | 商户地区 | 代码类 | 6!n |  | 参考行政区划通用填报要求。境内地区按照《中华人民共和国行政区划代码》（GB/T 2260）填写 6 位国内行政区划代码。 |  |

| B070017 | 采集日期 | 日期类 | YYYY-MM-DD |  | 数据表字段信息的采集日期，特定日历日的标识。 |  |

## 备注

- 报送对象覆盖线下 POS、线上终端和二维码商户。
- 同一商户存在多个 POS 终端时，按多条报送。
- 商户在报数期开始前已失效或注销的，不填报。
- `B070001 商户ID` 是商户唯一识别码，第三方支付机构收单交易也要填商户编号。
