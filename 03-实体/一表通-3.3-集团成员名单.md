# 3.3 集团成员名单

## 简介

3.3《集团成员名单》是监管一表通系统中的集团成员明细表，用于采集按集团客户授信风险管理口径识别的企事业法人成员，以及供应链融资纳入范围的成员。

## 关键信息

- 所属系统：[[03-实体/一表通系统|一表通系统]]
- 报表编码：3.3
- 报表名称：集团成员名单
- 整理日期：2026-04-14
- 来源状态：用户提供文本，待补正式制度文件来源与版本日期

## 当前结论

- 该表是集团成员关系明细表，不是集团主档表。
- 报送对象覆盖按照集团客户授信风险管理指引确定的企事业法人成员。
- 供应链融资场景下，填报银行纳入该项供应链融资范围的成员。
- 即使集团客户成员不是本机构客户，也应在本表填报。
- `成员类型` 区分核心成员和一般成员，核心成员是集团或供应链链条中的核心主体。
- `关系ID` 和 `采集日期` 共同构成主键。

## 字段结构

- C030001 关系ID
- C030002 成员ID
- C030003 成员企业名称
- C030004 成员统一社会信用代码
- C030005 成员类型
- C030006 登记注册代码
- C030007 集团ID
- C030008 机构ID
- C030009 关系状态
- C030010 采集日期

## 重点规则

- `C030002 成员ID` 为成员企业客户ID，每个成员需单独填列。
- `C030005 成员类型` 只有核心成员和一般成员两类。
- `C030007 集团ID` 用于回连集团主档。
- `C030009 关系状态` 只判断成员关系是否有效。
- `C030010 采集日期` 与 `C030001 关系ID` 一起构成主键。

## 规则线索

- `成员类型`、`关系状态`、`集团ID` 可沉淀为集团成员识别规则页。
- `成员统一社会信用代码`、`登记注册代码`、`成员企业名称` 可用于成员身份核对。
- 该表可与 `2.2 集团基本情况`、`3.1 重要股东及主要关联企业`、`3.2 高管及重要关系人信息` 共同构成集团关系网络。
- 该表也可与 `3.4 集团实际控制人` 联动，补足集团控制链。
