# 1.1 机构信息

## 简介

1.1《机构信息》是监管一表通系统中的机构主数据表，用于采集所有分配内部机构号的分支机构及内设机构基本信息，覆盖虚拟机构和理财子公司。

## 关键信息

- 所属系统：[[03-实体/一表通系统|一表通系统]]
- 报表编码：1.1
- 报表名称：机构信息
- 整理日期：2026-04-14
- 来源状态：用户提供文本，待补正式制度文件来源与版本日期

## 当前结论

- 该表核心功能是建立机构级主数据底座，而不是单纯记录机构名称。
- 机构识别链条以 `机构ID -> 内部机构号 -> 金融许可证号` 为主线，兼顾统一社会信用代码、支付行号和行政区划。
- `机构类别` 与 `机构类型` 分别描述组织形态和金融业态，二者不能混用。
- `县域机构标识`、`科技支行标识`、`科技特色支行标识`、`科技金融专营机构标识`、`自贸区网点标识` 是对机构属性的专项标记。
- `采集日期` 是关键主键字段之一，说明该表更接近按日快照的机构台账。

## 重点规则

- A010001 的生成规则：前 11 位取金融许可证号前 11 位，后 13 位取内部机构号，不足不补齐。
- 理财登记中心填报机构ID统一使用金融许可证。
- 若发行地涉及多个国家，机构ID填写共同上级行机构ID，以支持多地区填报。
- A010003 若本机构没有金融许可证，则取上一级机构的金融许可证号；内设机构、虚拟机构及营业部取上一级管理机构金融许可证号。
- A010008 的机构类别区分管理机构、营业机构、虚拟机构和内设机构，是后续统计和路由判断的重要依据。
- 县域认定以机构注册地是否位于县域地区为主要依据，县域名单口径参照 2020 年行政区划代码。
- A010015 默认值为 `9999-12-31`，表示未发生或待确认的成立日期。
- 境内涉密机构在 A010016 不填报地址。

## 规则线索

- `机构ID`、`金融许可证号`、`内部机构号` 之间存在稳定生成关系，后续可单独沉淀为机构编码规则页。
- `县域机构标识`、`自贸区网点标识`、`科技支行标识`、`科技金融专营机构标识` 可能适合沉淀为机构属性规则页或维表页。
- `机构层级` 与 `机构类别` 适合做成分类对照，避免与 `机构类型` 混用。
