# 1104-G33_Ⅰ-银行账簿利率风险计量报表（标准化计量框架简化版）

## 简介

G33_Ⅰ 是 1104 监管报表体系中的**银行账簿利率风险（IRRBB）**计量报表简化版，与 G33_Ⅱ（完整版）同依据《商业银行银行账簿利率风险管理指引（修订）》，但适用于**大型行/股份制行以外**的机构（城商行、农商行、政策性银行、外资法人行等）。两者框架相似，核心区别在于**行为性期权无需建模**、**无到期日存款直接填隔夜**、**利率期权不作独立经济价值计算**，大幅降低了填报复杂度。

## 关键信息

- 所属系统：[[03-实体/1104系统|1104系统]]
- 报表编码：G33_Ⅰ
- 报表名称：银行账簿利率风险计量报表（标准化计量框架简化版）
- 整理日期：2026-04-13
- 来源状态：填报说明原文已入库
- 政策依据：原银保监会《商业银行银行账簿利率风险管理指引（修订）》
- 适用机构：政策性银行（含开发银行）、城市商业银行、民营银行、农村商业银行、外资法人银行
- 数据单位：万元，金额保留两位小数
- 报送口径：法人口径，分主要货币填报
- 频率：季报
- 截止日：季后18日（第二批次）

## 当前结论

- **机构互补**：G33_Ⅰ 与 G33_Ⅱ 共同覆盖全部银行机构，两张报表的适用机构不重叠——大型行+股份制行填 G33_Ⅱ，其余机构填 G33_Ⅰ
- **行为性期权无需建模**：具备提前还款权的贷款（[1.2]/[1.3]）和可提前支取的存款（[2.3]/[2.4]）**直接按合同约定现金流**填报，无需计算不同利率情景下的提前行为率；这是与 G33_Ⅱ 最核心的简化
- **无到期日存款一律填隔夜**：[2.2] 全部填入【隔夜】区间（A=B），无需10年历史数据建模识别核心/非核心存款；[2.2.1核心存款] 暂不填报
- **利率期权不作独立经济价值计算**：G33_Ⅰ 的 [5.利率期权] 仅登记头寸（多头/空头），无 G33_Ⅱ 中的 [7.利率期权经济价值计算] 复杂计量模块；债券中的嵌入式期权**无需剥离**（G33_Ⅱ 需要）
- **NII冲击幅度表述不同**：G33_Ⅰ 使用"标准化利率冲击幅度"（随币种和期限结构确定），G33_Ⅱ 明确为"250个基点"
- **项目编号差异**：G33_Ⅰ 的[5.]=利率期权、[6.]=无风险收益率曲线、[7.]=k值；G33_Ⅱ 的[5.]=无风险收益率曲线、[6.]=k值、[7.]=利率期权经济价值计算、[11.]=利率期权头寸

## 重点规则

### G33_Ⅰ vs G33_Ⅱ 核心差异对照

| 维度 | G33_Ⅰ（简化版） | G33_Ⅱ（完整版） |
|------|----------------|----------------|
| 适用机构 | 政策性行、城商行、农商行、民营行、外资法人行 | 大型行（含邮储）、股份制行 |
| 行为性期权（提前还款/支取） | 直接按合同现金流填报，无需建模 | 须建模计算6种情景下的提前行为率并调整现金流 |
| 无到期日存款 | 全部填【隔夜】（A=B），[2.2.1]暂不填报 | 须用10年历史数据区分核心/非核心，有监管上限 |
| 嵌入式利率期权 | 无需剥离，连同基础工具一起填[1.]/[2.] | 须剥离，期权部分独立填[11.]+[7.] |
| 利率期权计量 | [5.] 仅登记头寸（多头/空头），无独立经济价值计算 | [11.]+[7.] 须计算每种冲击情景下的期权价值变动，波动率×1.25 |
| NII冲击幅度 | "标准化利率冲击幅度"（由监管框架确定） | 明确为"250个基点" |
| [5.] 对应项目 | 利率期权 | 无风险收益率曲线 |
| [6.] 对应项目 | 无风险收益率曲线 | k值 |
| [7.] 对应项目 | k值 | 利率期权经济价值计算（复杂） |
| [11.] | 不存在 | 利率期权头寸登记 |

### 无到期日存款的特殊列校验

[2.2] 的填报特点：
- 全部填入A列（即B列）=【隔夜】区间，其余时间段为0
- 列校验规则专门设定：A=B（而非A=B至T合计），反映了"一律填隔夜"的要求
- [2.2.1核心存款] 暂不填报——预留字段，将来可能启用

### 行为性期权的简化处理

G33_Ⅰ 中 [1.2]、[1.3]、[2.3]、[2.4] 均**直接填合同约定现金流**，不区分不同利率情景。这意味着：
- 不需要估算提前还款概率/提前支取概率
- 不需要区分基准情景和6种冲击情景下的现金流差异
- 但填报结果对利率风险的测量精度相对较低（适配中小机构的管理能力）

### 重定价日期规则（与G33_Ⅱ完全相同）

| 产品类型 | 本金重定价日 | 利息重定价日 |
|---------|------------|------------|
| 固定利率 | 合同到期日 | 每个利息支付日 |
| 浮动利率 | 距报告日最近的利率调整日 | 所有未偿还/未重定价本金的利息支付日 |
| 利率调整日不确定 | 紧接报告日的工作日 | 同上 |
| 逾期资产（非非应计） | 填入【隔夜-1个月】区间 | — |

利率触及上下限：视同固定利率，重新浮动后再按浮动利率处理。

### 衍生品复式记录（与G33_Ⅱ完全相同）

参照 G4C-1(b) 填报规则。

## 规则线索

- **G33_Ⅰ/Ⅱ 的机构分层设计**：简化版/完整版的差异本质是监管对不同规模机构风险管理能力的差异化要求；大型行/股份制行须精细建模，中小机构用保守简化假设（行为期权不建模=视同合同期限，活期存款视同隔夜到期），两种处理均会导致对IRRBB的高估——监管保守设计
- **[2.2.1核心存款"暂不填报"的信号**：这是监管预留字段，未来可能要求中小机构也开始对活期存款建模识别核心存款；与G33_Ⅱ的 [2.2.1]-[2.2.3] 对比，可以预判监管收严的方向
- **NII幅度"标准化"vs"250bps"的差异**：G33_Ⅰ 使用监管框架确定的标准化幅度（不同币种不同期限档位可能不同），G33_Ⅱ 明确为250bps；两张报表的NII计量结果不直接可比，跨机构类型比较时需注意口径差异
- **与G4C-1(b)的衍生品填报衔接**：G33_Ⅰ/Ⅱ 对衍生品复式记录的处理逻辑均参照 G4C-1(b)，说明利率风险报表与市场风险资本报表在衍生品分类上保持一致，G4C系列入库后可作为G33系列衍生品填法的参照来源
