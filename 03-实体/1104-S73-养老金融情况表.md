# 1104-S73-养老金融情况表

## 简介

S73《养老金融情况表》是 1104 监管报表体系中的养老金融专项报表，编码为"金监统S73"（国家金融监督管理总局），季报第一批次报送。表分三部分：养老产业贷款、养老金金融、养老服务金融。

## 关键信息

- 所属系统：[[03-实体/1104系统|1104系统]]
- 报表编码：金监统S73
- 报表名称：养老金融情况表
- 整理日期：2026-04-11
- 来源状态：填报说明原文已入库

## 当前结论

- S73 是养老金融综合统计口径，横跨贷款（产业端）、养老金（资金端）、财富管理与长辈服务（客群端）三个维度
- 养老产业贷款认定采用"50% 占比门槛"：实际投向≥50% 计入；无法判断时以借款人上年营收中养老收入≥50% 为准
- 与 G01_Ⅶ（贷款投向 [8.A/B]）、G01_Ⅲ（存贷款明细 [6.C]）、G06a_I（非保本理财）形成表间核对，是跨报表关联最多的专项表之一
- 季报第一批次（季后13日），早于 G14 系列的季后18日
- 企业集团财务公司、信托公司、理财公司均需填报（与 G14_II/IV 的适用机构范围有差异）
- 数据单位混用：金额为万元，账户/卡片为户/人，产品数量为支

政策性银行（含开发银行）、大型银行（含邮储银行）、股份制商业银行、城市商业银行、农村商业银行、村镇银行、外资法人银行、外国银行分行、民营银行、直销银行、农村合作银行、农村信用合作社、企业集团财务公司、金融租赁公司、信托公司、理财公司。

### 第Ⅰ部分：养老产业贷款

| 字段 | 说明 |
|------|------|
| `[A]` 当年累放贷款户数 | 年初至期末累计发放过贷款的客户数（含年初有余额且年中再贷款的客户） |
| `[B]` 当年累计发放贷款金额 | 年初至期末累计发放金额 |
| `[C]` 贷款余额 | 含透支、经营贷款、固定资产贷款、融资租赁、贸易融资、并购贷款、买入返售、垫款、票据融资（直贴） |
| `[D]` 中长期贷款余额 | 原始期限一年以上 |
| `[E]` 不良贷款余额 | 次级+可疑+损失 |
| `[F]` 当年累放贷款年化利息收益 | 当年发放贷款的利息收入按年折算，**往年发放贷款利息不计** |

**认定规则（行）**：

| 行              | 说明                                                                                 |
| -------------- | ---------------------------------------------------------------------------------- |
| `[1]` 养老产业贷款   | 按《养老产业统计分类（2020）》（国家统计局令第30号）认定；以每笔借据为标准，实际投向≥50% 计入；无法判断时，借款人上年度财务报表养老收入占比≥50% 计入 |
| `[2]` 全国养老机构贷款 | 借款人须在民政部"民政一体化政务服务平台"发布的全国养老机构名单内                                                  |

| 职业年金 | `[5.C]` 受托管理规模 | 作为受托管理人，以市值计的职业年金基金总额 |

### 第Ⅲ部分：养老服务金融

| 项目 | 说明 |
|------|------|
| `[7.1]` 特定养老储蓄存款 | 依原银保监会《特定养老储蓄产品试点操作规程》吸收，产品需标识"特定养老储蓄产品"字样，与 G01_Ⅲ[6.C] 勾稽 |
| `[7.2.1]` 养老理财产品 | 符合养老理财监管要求的理财产品，与 G06a_I[7.3] 勾稽 |
| `[7.2.2]` 个人养老金理财产品 | 符合个人养老金理财监管要求，与 G06a_I[7.4] 勾稽；≥[7.2.2.1] |
| `[7.2.2.1]` 通过个人养老金持有的养老理财产品 | 通过个人养老金资金账户持有，与 G06a_I[7.4.3] 勾稽 |
| `[8]` 代销养老金融产品 | 年初至期末累计代销：养老理财、养老目标基金、商业保险年金等，统计产品数量和规模 |
| `[9.1]` 长辈客群存款 | 年龄≥55岁客户持有的各项存款 |
| `[9.2]` 长辈客群贷款 | 年龄≥55岁客户的个人贷款 |
| `[9.3]` 长辈客群代理代销 | 年龄≥55岁客户的代销产品（范围同G01_I[20.代理代销业务]）：信托、资管、保险、基金、理财 |

## 重点规则

### 表内校验关系

```text
[7.2.2] ≥ [7.2.2.1]        （适用于A、B列，个人养老金理财包含通过养老金持有的部分）
[C] ≥ [D]                   （贷款余额 ≥ 中长期贷款余额）
[C] ≥ [E]                   （贷款余额 ≥ 不良贷款余额）
```

### 表间校验关系

```text
[1.C] = G01_Ⅶ[8.A]          （养老产业贷款余额 → 贷款投向附注）
[1.D] = G01_Ⅶ[8.B]          （养老产业中长期贷款 → 贷款投向附注）
[7.1B] = G01_Ⅲ[6.C]         （特定养老储蓄 → 存贷款明细存款项）
[7.2.1A] = G06a_I[7.3D]      （养老理财产品数量 → 非保本理财月报）
[7.2.1B] = G06a_I[7.3E]      （养老理财产品规模 → 非保本理财月报）
[7.2.2A] = G06a_I[7.4D]      （个人养老金理财产品数量）
[7.2.2B] = G06a_I[7.4E]      （个人养老金理财产品规模）
[7.2.2.1A] = G06a_I[7.4.3D] （通过养老金持有的理财产品数量）
[7.2.2.1B] = G06a_I[7.4.3E] （通过养老金持有的理财产品规模）
```

## 规则线索

- **养老产业认定的"50%门槛"**：与贷款实际投向判断方法（G01_Ⅶ 行业分类）存在关联，后续可建立"贷款投向认定"规则页
- **"三支柱"养老金体系**：本表体现了第一支柱（基本养老保险）、第二支柱（企业/职业年金）、第三支柱（个人养老金）的完整覆盖
- **G06a_I 的多处勾稽**：本表对理财产品的5个字段均需与 G06a_I 保持一致，G06a_I 尚未入库，后续引入时需补充此勾稽关系
- **长辈客群统计口径（55岁）**：监管视角的老龄客群分界线，与普通零售客群有别，后续可考虑建立"个人客群分类"规则线索
