# LLM Wiki Agent Guide

本仓库是一个由 LLM 协助维护的个人知识库，目标不是临时回答问题，而是持续沉淀结构化知识。

## 核心原则

1. 原始资料只增不改，作为事实来源保留。
2. Wiki 页面是整理后的知识层，允许持续更新。
3. 每次处理新资料时，优先更新已有页面，而不是重复创建近义页面。
4. 回答用户问题时，优先基于 Wiki 页面组织答案，必要时再回溯原始资料。
5. 重要的新结论、新对比、新框架，应回写到 Wiki，而不是只停留在聊天记录里。

## 目录约定

- `00-首页/`：入口页、总索引、工作说明。
- `01-资料库/`：原始资料的摘录、导入文本、来源记录。
- `02-主题/`：按主题组织的核心知识页。
- `03-实体/`：人物、公司、产品、项目、概念对象等实体页。
- `04-综合/`：综述、对比、路线图、FAQ、决策文档。
- `04-综合/梦境/`：系统对“你最近在变化什么”的定期综合页。
- `05-日志/`：摄入、问答、巡检日志。
- `06-项目/`：按项目维度维护的长期页面。
- `90-模板/`：页面模板。

## 页面类型

### 资料页

路径：`01-资料库/`

用途：
- 保存原始文章的摘要、来源、关键摘录
- 作为后续更新主题页和实体页的依据

建议文件名：
- `YYYY-MM-DD-标题.md`

建议包含：
- 来源链接
- 作者/发布时间
- 三到五条核心观点
- 可更新哪些主题页/实体页
- 与已有页面的关联

### 主题页

路径：`02-主题/`

用途：
- 汇总某个主题下的稳定知识
- 不写成一次性摘要，而写成长期维护页面

建议包含：
- 主题定义
- 当前结论
- 关键争议
- 关联实体
- 关联资料
- 待补充问题

### 实体页

路径：`03-实体/`

用途：
- 针对具体对象建立持续更新档案

适用对象：
- 人物
- 公司
- 产品
- 模型
- 论文
- 方法论名词

### 综合页

路径：`04-综合/`

用途：
- 沉淀较高价值的跨页成果

常见类型：
- 对比分析
- 研究综述
- 学习路径
- 决策备忘
- 问题清单

### 梦境页

路径：`04-综合/梦境/`

用途：
- 生成系统对“你最近在做什么、变成什么、该往哪里走”的阶段性理解
- 作为个人上下文的中间层，不直接覆盖稳定自我页

建议包含：
- 本次主题
- 观察
- 推断
- 可能忽视的信号
- 建议更新的核心页
- 下一步问题
- 一句话画像

## 三类操作

### Ingest

当用户添加新资料时：

1. 创建或更新资料页到 `01-资料库/`
2. 提炼核心观点
3. 找到受影响的主题页和实体页
4. 将新信息合并进对应页面
5. 补充交叉链接
6. 在 `05-日志/log.md` 追加记录

如果资料与现有内容冲突：
- 明确标记冲突点
- 不静默覆盖旧结论
- 在相关页面注明依据来源

### Query

当用户提问时：

1. 先查看 `00-首页/index.md`
2. 再查看相关主题页、实体页、综合页
3. 必要时回溯资料页
4. 输出回答时区分“已知结论”和“推断判断”
5. 如果回答产生了长期有价值的新内容，回写到 `04-综合/` 或相关页面
6. 在 `05-日志/log.md` 记录本次问答

### Dream

当用户要求“做一次梦”时：

1. 优先读取 `00-首页/index.md`
2. 读取 `00-首页/我是谁.md`、`00-首页/我的偏好.md`、`00-首页/我的目标.md`
3. 读取 `06-项目/当前学习地图.md`、相关项目页和 `05-日志/log.md`
4. 优先关注最近 7 天更新过的资料页、主题页、综合页；如信息不足可回看最近 30 天
5. 输出时必须区分“观察”和“推断”
6. 推断允许大胆，但不能写成既定事实
7. 自动落到 `04-综合/梦境/YYYY-MM-DD-梦境.md`
8. 只提出建议更新的核心页，不直接改写 `我是谁`、`我的偏好`、`我的目标`、`当前学习地图`
9. 在 `05-日志/log.md` 记录本次 dream

### Lint

定期巡检知识库：

- 是否有重复页面
- 是否存在孤立页面
- 是否缺少重要交叉链接
- 是否有明显过时结论
- 是否有频繁出现但尚未建页的概念
- 是否有冲突信息未标注来源

巡检后：
- 优先修补结构问题
- 再在 `05-日志/log.md` 记录本次巡检

## 写作约定

1. 优先使用中文。
2. 页面标题简洁明确，避免同义重复。
3. 一页只承载一个稳定主题。
4. 使用 Markdown 链接互相引用。
5. 不堆砌长摘录，优先写提炼后的结论。
6. 对不确定内容显式标注“待确认”。
7. 对时间敏感内容尽量写明具体日期。

## 命名建议

- 主题页：`主题名.md`
- 实体页：`实体名.md`
- 综合页：`主题-对比.md`、`主题-综述.md`、`主题-路线图.md`
- 资料页：`YYYY-MM-DD-标题.md`

## 每次开始工作时优先读取

1. `00-首页/index.md`
2. `05-日志/log.md`
3. 与当前任务最相关的页面

## 每次结束工作时检查

1. 是否更新了受影响页面
2. 是否补上了交叉链接
3. 是否把高价值输出沉淀成页面
4. 是否记录到日志
