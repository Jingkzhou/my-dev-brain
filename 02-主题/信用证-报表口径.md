# 信用证-报表口径

## 概念定义

1104 口径下，“信用证业务”至少要区分 5 种常见状态：

- **开立跟单信用证但尚未融资/垫款**：表外担保承诺业务
- **收取信用证保证金**：负债端保证金存款
- **信用证项下融资**：如进口押汇、出口押汇、议付信用证，进入贸易融资
- **信用证发生垫款**：进入各项垫款/各项贷款口径
- **信用证相关资产转让**：如已议付信用证（押汇）转让，进入买断其他票据类资产口径

因此，“做了一笔信用证业务”不能只看一个表，要先看这笔业务现在处于哪个状态。

## 直接回答

### 情形一：只是“开了一笔信用证”，尚未融资、尚未垫款

这时最直接应填的 1104 报表是：

- `G01_I_1[2.跟单信用证]`
- `G21` 第Ⅲ部分 `[13.发行的跟单信用证]`

如果同时收了保证金，还要填：

- `G01[34.存入保证金]`
- `G01_III` 中对应的单位/个人保证金存款口径

### 情形二：信用证项下已经发生融资

例如进口押汇、出口押汇、议付信用证，这时应转入贸易融资口径：

- `G01[7.贸易融资]`
- `G01_III[1.4贸易融资]`
- 如果是外币，还要看 `G01_V[5.贸易融资]`
- `G21[1.6各项贷款]` 按剩余期限分桶

### 情形三：信用证项下已经发生垫款

这时不再只是“表外开证”，而是进入垫款/贷款化口径：

- `G01_III[1.6各项垫款]`
- 汇总进入 `G01[62.各项贷款]`
- 同时进入 `G21[1.6各项贷款]`

### 情形四：这笔信用证对应企业客户专项统计

如果客户属于专项报表覆盖范围，还可能同步进入：

- `S63[4.2跟单信用证]`：企业融资总量表
- `S63_III[9.跟单信用证]`：控股类型切片
- `S72[8.跟单信用证]`：制造业客户

### 一句话判断

- **开证未融资**：先看 `G01_I_1 + G21`
- **收保证金**：再看 `G01 + G01_III`
- **信用证项下融资**：再看 `G01 + G01_III + G01_V + G21`
- **信用证已垫款**：再看 `G01_III各项垫款 + G01各项贷款 + G21`
- **客户落在专项范围**：再补 `S63/S63_III/S72`

## 易混淆点

1. **“跟单信用证”与“议付信用证”不是同一层口径**
   - 跟单信用证本身是表外承诺，主看 `G01_I_1[2]`
   - 议付信用证款项是融资，主看 `G01[7]`、`G01_III[1.4]`

2. **“信用证”与“信用证垫款”不是同一层口径**
   - 前者是表外业务
   - 后者是代客户垫付资金后的表内风险暴露

3. **保证金不是信用证余额**
   - 信用证保证金走负债端保证金存款，不并入信用证余额或贸易融资余额

4. **已议付信用证转让不继续填贸易融资**
   - 在 G01_Ⅶ 中，它更接近“买断其他票据类资产”

5. **专项表是“专项 ≤ 全口径”**
   - S63/S63_III/S72 中的跟单信用证，本质上是对 `G01_I_1[2]` 的客户维度切片，不是额外新增一套独立口径

## 已知结论与推断

- 已知结论：
  - 开立的跟单信用证直接进入 `G01_I_1[2]`
  - G21 第Ⅲ部分明确要求填 `[13.发行的跟单信用证]`
  - 议付信用证属于贸易融资
  - 信用证垫款属于各项垫款
  - 已议付信用证转让进入买断其他票据类资产
- 推断判断：
  - 如果用户只说“做了一笔信用证业务”，在没有补充“是否已押汇/议付/垫款”的情况下，**最稳妥的默认回答应先落在“开证未融资”口径**，即 `G01_I_1[2] + G21第Ⅲ部分[13]`；其余报表取决于后续是否收保证金、是否发生融资、是否发生垫款、是否进入专项统计范围。
